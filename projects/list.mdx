# List Projects

Retrieve a paginated list of all projects, sorted by name.

## API Request

<Card title="GET All Projects" icon="building">
  ```bash
  GET /api/v1/initiatives/projects
  ```
</Card>

## Query Parameters

| Parameter | Type | Description | Required | Default | Possible Values |
|-----------|------|-------------|----------|---------|-----------------|
| implementing_agency | String | The unique identifier of the implementing agency (UUID) | No | None | Valid agency UUID |
| locations | String | Location code(s) to filter | No | None | Single: "A01", "A02"<br/>Multiple: "A01,T01,T02" |
| functions | String | Initiative function(s) to filter by | No | None | Single: "water-drainage-system"<br/>Multiple: "water-drainage-system,port" |
| page | Integer | The page number to retrieve | No | 1 | Any positive integer |
| per_page | Integer | The number of items per page | No | 20 | Min: 1, Max: 50 |

## Example Requests

### Filter by Single Location
```bash
GET /initiatives/projects?locations=A01
```

### Filter by Multiple Locations
```bash
GET /initiatives/projects?locations=A01,A02,A03
```

### Set Items Per Page
```bash
GET /initiatives/projects?per_page=20
```

### Combined Filters with Pagination
```bash
GET /initiatives/projects?locations=A01,T01&functions=water-drainage-system,port&per_page=20&page=1
```

      Location name
    </ResponseField>

        <ResponseField name="type" type="string">Agency type</ResponseField>
      </Expandable>

  </Expandable>
</ResponseField>

## Notes

- Projects are sorted by name in ascending order
- The response includes agency details without business area information
- Location codes in the locations parameter are validated for accuracy
